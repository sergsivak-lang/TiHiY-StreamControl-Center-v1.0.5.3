# TiHiY StreamControl Center v1.0.3 — внутрішня карта реалізації

Цей файл фіксує затверджені правила інтерфейсу, щоб наступні зміни не ламали вже готовий макет.

## 1. Базова тема

- Cyber Blue + Amber.
- Темний фон, cyan-контури, amber-активні дії.
- Фіксовані декоративні кути, які не розтягуються разом із кнопкою або панеллю.
- Кольорові теми додаються тільки після стабілізації функціоналу.

## 2. Єдина система кнопок

- `Button` — базова адаптивна cyber-кнопка.
- `PrimaryActionButton` — верхні основні дії.
- `ModuleButton` — сітка модулів.
- `ModuleAmberButton` — активний/донатний модуль.
- `UtilityButton` — кнопки дочірніх вікон.
- `WindowButton` — згорнути, розгорнути, закрити.
- Вміст кнопки розміщується в `Viewbox` із `StretchDirection=DownOnly`: текст та іконка зменшуються тільки тоді, коли їм бракує місця, а не обрізаються.
- Заборонено повертати геометрію кнопок із фіксованим полотном 1000×500.

## 3. Іконки платформ

Ресурси: `Assets/Platforms/`.

- Twitch: статус, чат, кнопки відправлення, підписки/Bits, вікно каналів.
- YouTube: статус, чат, трансляція, Super Chat, YouTube Studio.
- Discord: канали, бот, сповіщення, відповідні дочірні вікна.
- Donatello: донати, тестова подія, модуль і вікно Donatello.
- OBS: підключення та OBS-дії.

Іконки є WPF Resource і використовуються через pack URI, тому не залежать від робочої папки EXE.

## 4. Адаптивний макет

- Головна Grid завжди займає реальний доступний розмір вікна.
- Заборонено застосовувати до неї загальний `LayoutTransform` або подвійний масштаб.
- Auto scale змінює успадкований розмір шрифту в межах 82–118%.
- Кнопки та контент природно перебудовуються сіткою; за нестачі ширини їхній вміст стискається без обрізання.
- Мінімальний розмір головного вікна: 1180×760.

## 5. Перетягування блоків

Окремі розділювачі керують:

1. шириною мультичату та швидкого OBS-мікшера;
2. висотою верхнього і нижнього ряду;
3. шириною блоку донатів та блоку модулів незалежно від верхнього ряду;
4. висотою нижнього інформаційного ряду;
5. шириною «Стан системи», «Останні події» та «Моніторинг ПК».

Пропорції зберігаються в `settings.json`. Подвійний клік по будь-якому розділювачу повертає стандартний макет. Кожен блок має мінімальну ширину/висоту, нижче якої його не можна стиснути.

## 6. Дані та функціональність

- Після підключення сервісу відображаються тільки його реальні дані.
- Без підключення показується чесний empty state, а не демонстраційні значення.
- TEST-події мають явну позначку та не змінюють реальну ціль збору.
- Зміни UI не повинні підміняти або відключати робочу логіку OBS, Twitch, YouTube, Discord, Donatello, музики, overlay, журналу та AIDA64.
