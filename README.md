# TiHiY StreamControl Center v1.0.5.2 — Responsive Functional UI

Затверджена базова версія у темі **Cyber Blue + Amber** з еталонною темою **Україна**. У звичайному режимі головна панель не використовує демонстраційні чат-повідомлення, аудіоканали або донати: блоки показують реальні дані підключених сервісів або чесний стан «не підключено».

## Працюючі модулі

- **OBS WebSocket v5:** підключення, реальні аудіоджерела, live meters, гучність, Mute, сцени, Preview/Program, запис, трансляція та статистика OBS.
- **Twitch:** OAuth, IRC-чат, надсилання повідомлень, live-статистика, модерація, subscriptions/bits.
- **YouTube:** OAuth Desktop App, пошук активного ефіру, live chat, надсилання, модерація, Super Chat/Super Sticker/members, налаштування трансляції.
- **Donatello:** офіційний X-Token API, нові донати й платні підписки, імпорт, ціль збору, історія, alert replay та окремий Donatello overlay (ціль збору + останній донат + сповіщення).
- **Discord:** повідомлення про старт Twitch/YouTube та окремі monetization-події в задані канали.
- **Overlay Server:** chat, alerts і now-playing для OBS Browser Source.
- **Музика:** локальний плейлист і Now Playing overlay.
- **Журнал і діагностика:** події, помилки, startup crash logs, CPU/RAM/OBS FPS/мережа/uptime.

## Правила даних

- Після підключення показуються тільки реальні дані сервісів.
- Кнопка **TEST-ПОДІЯ** створює подію з позначкою TEST. Вона не змінює реальну ціль збору і не надсилається в Discord.
- Повтор alert не додає суму вдруге до цілі збору.
- Секрети зберігаються через Windows Credential Manager.

## Запуск

1. Повністю розпакуйте ZIP у нову папку.
2. Встановіть **Microsoft .NET 9 SDK x64**.
3. Запустіть `START-HERE.cmd`.
4. Після складання програма буде у папці `Release`. Не відокремлюйте EXE від DLL-файлів.

`START-HERE.cmd` і `BUILD-AND-RUN-CLEAN.cmd` автоматично читають актуальну назву збірки з `VERSION.txt`, тому повідомлення запуску та журнали відповідають поточному релізу.

## Перші налаштування

1. У OBS: **Інструменти → Налаштування WebSocket Server**, увімкнути сервер на порту `4455`.
2. У програмі відкрити **Канали** та авторизувати Twitch/YouTube.
3. У модулі **Donatello** зберегти API Token.
4. У модулі **Сповіщення Discord Bot** зберегти Bot Token та ID каналів.
5. Додати OBS Browser Source:
   - `http://127.0.0.1:17845/overlay/chat`
   - `http://127.0.0.1:17845/overlay/alerts`
   - `http://127.0.0.1:17845/overlay/donatello`
   - `http://127.0.0.1:17845/overlay/now-playing`

## Перевірка складання

У репозиторії є GitHub Actions для Windows x64. Workflow збирає self-contained пакет і запускає WPF у спеціальному CI screenshot-режимі. Демонстраційні дані існують лише в цьому явному CI-режимі й не використовуються під час звичайного запуску.

## Системний монітор v1.0.5.2

Нижній правий блок показує тільки реальні показники:

- поточний час;
- частоту CPU та оперативної пам’яті;
- завантаження CPU і GPU;
- температуру CPU і GPU;
- використання RAM і VRAM.

AIDA64 Shared Memory читається як потік XML-сенсорів без вимоги одного кореневого елемента. Це виправляє помилку `There are multiple root elements`. Без AIDA64 CPU load, RAM і мережа беруться з Windows, а недоступні датчики чесно показуються як `—`.

## Адаптивні кнопки та іконки платформ

- усі головні, модульні й службові кнопки використовують одну Cyber Blue + Amber систему з фіксованими декоративними кутами;
- текст та іконка всередині кнопки зменшуються за нестачі місця замість обрізання;
- Twitch, YouTube, Discord, Donatello та OBS використовують впізнавані PNG-іконки з `Assets/Platforms`;
- дочірні вікна успадковують компактну версію того самого стилю;
- кнопки керування вікном більше не вимикають власну рамку локальними перевизначеннями.

## Масштабування й розміри блоків

- у режимі **АВТО** блоки, кнопки та шрифти масштабуються разом зі зміною розміру головного вікна;
- вертикальні та горизонтальні світні розділювачі дозволяють міняти ширину й висоту блоків мишкою;
- макет зберігається у `settings.json` і відновлюється після запуску;
- подвійний клік на розділювачі повертає стандартні пропорції.

Інструкція AIDA64: `AIDA64-SETUP-UA.txt`.
